# Política de seguridad

## Versiones soportadas

Este proyecto se mantiene bajo un esquema de mejor esfuerzo.

| Versión / rama | Soporte |
|----------------|---------|
| Rama principal | ✅ Sí |
| Última versión publicada | ✅ Mejor esfuerzo |
| Ramas viejas o forks | ❌ No |

## Cómo reportar una vulnerabilidad

NO abras issues públicos para reportar problemas de seguridad.

Seguí este proceso:

1. Si GitHub habilita reporte privado de vulnerabilidades para este repo, usalo.
2. Si no está disponible, contactá al equipo por un canal privado antes de divulgar nada públicamente.
3. Explicá el problema con claridad: impacto, partes afectadas y pasos para reproducir.
4. Si ya tenés una mitigación o idea de solución, sumala.

## Qué incluir en el reporte

Mientras más contexto, mejor:

- tipo de vulnerabilidad
- archivo, módulo o componente afectado
- condiciones necesarias para explotarla
- prueba de concepto o pasos de reproducción
- impacto posible
- propuesta de remediación, si la conocés

## Qué podés esperar de nuestra parte

Vamos a intentar:

- confirmar recepción dentro de 5 días hábiles
- evaluar severidad e impacto
- coordinar corrección y plan de divulgación
- reconocer a quien reportó, si corresponde y lo desea

## Divulgación responsable

- Danos tiempo razonable para investigar y corregir antes de hacerlo público
- Evitá publicar detalles explotables mientras la solución no esté lista
- Si afecta usuarios o despliegues, coordinemos el momento de divulgación
